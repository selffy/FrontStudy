
웹개발

웹브라우저 : 인터넷 사용자에게 데이터를 전달해주는 매개 역할

서버 vs 클라이언트


차이점(기준): 웹브라우저-클라이언트, 자바/데이터베이스-서버
클라이언트: 프런트엔드 / 웹브라우저
서버: 백앤드 / 웹 요소와 여러 정보가 저장되는 곳
	예) 안드로이드 등

클라이언트		<->		서버
----------------->        <-----------------
웹브라우저 통해			요청을 처리해
필요한 정보 요청			알맞은 정보 전달

프런트엔드: 요청을 받는 곳, 요청을 확인하는 곳(시각적 확인)
	예) HTML, CSS, 자바스크립트 등
백앤드: 요청을 처리하는 곳, 보이지 않은 영역(서버)을 다룸.
	예) 자바, PHP, 파이썬 등


<웹개발을 위해 공부해야할 기술>

HTML  ㅡ
		ㅣ	프런트엔드
CSS	    ㅡ
자바스크립트 ㅡ
			  ㅣ 백엔드
깃/깃허브	ㅡ

<웹 개발의 기본>
HTML:웹 문서 뼈대
CSS: 웹 인테리어
자바스크립트: 웹 기능

 :해당파일의 파일을 지정하는 곳
